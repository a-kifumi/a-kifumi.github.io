# プロジェクト追加ポリシー

最終更新: 2026-07-21

## 1. 目的

このサイトは、akifumi が実際に作成・運営するプロジェクトを訪問者が理解し、適切な公式サイトやストアへ移動できるようにするポートフォリオである。検索や外部サイトから自然に発見・参照されることは重要だが、Domain Rating（DR）、リンク評価、検索順位の操作を主目的にしない。

判断基準は「この追加がリンクなしでも訪問者に価値を提供するか」である。答えが No なら公開しない。

## 2. 追加前の必須ゲート

次のすべてを満たすまで、`index.html`、`ja/index.html`、個別ページを編集しない。

- [ ] akifumi が作成または継続的に運営する実在プロジェクトである。
- [ ] ローカルリポジトリ、公開サイト、App Store など一次情報を確認した。
- [ ] 名称、URL、提供状態、主要機能、対象利用者を一次情報と照合した。
- [ ] 掲載理由を「リンクを増やすため」以外の利用者価値で一文説明できる。
- [ ] 既存掲載物と実質的に重複せず、サイトの制作実績という主題に合う。
- [ ] 英語・日本語の双方に、自然で正確な説明を用意できる。
- [ ] リンク先が公開され、HTTPS で正常に開き、誤認を招く転送をしない。

一つでも満たせない項目があれば、推測で埋めずに追加を保留する。

## 3. 掲載形式の選択

### 一覧カードで十分な場合

プロジェクト名、独自の短い説明、適切なカテゴリ、公式の遷移先を示せば目的を達成できる場合は、トップページのカードだけにする。プロジェクト数を増やすための個別ページは作らない。

### 個別ページを作成できる場合

個別ページは、次の独自情報を訪問者に提供できる場合に限る。

- 何を解決するプロジェクトか、誰のためのものか
- 実際の画面を示すスクリーンショットまたはデモ画像と、内容を表す `alt`
- 作成背景、設計上の工夫、技術スタック、プライバシー方針などから、少なくとも二つの有用な説明
- 現在の提供状態と、利用できる公式サイト、App Store、リポジトリ等への明確な導線
- 他ページの言い換えや転載ではない、固有の本文

文字数だけで品質を判定しない。100〜200文字を機械的に水増しするより、短くても固有で検証可能な説明を優先する。

## 4. 禁止事項

次の変更は行わない。

- 検索順位やリンク評価の操作を主目的に、プロジェクト、ページ、リポジトリ、リンクを作ること
- 内容の薄いページ、ほぼ同一のページ、キーワード・地域だけを差し替えたページの量産
- 実際のツールより手前に、検索流入を転送するためだけの誘導ページを置くこと
- キーワードの不自然な反復、説明と無関係な検索語、隠しテキスト、隠しリンク
- 同一リンクの過剰な反復、キーワードを詰めた完全一致アンカー、全ページ共通部分への不必要な外部リンク埋め込み
- 被リンクとの交換を条件にした相互リンク、低品質ディレクトリへの大量登録、自動リンク生成
- 他サイトやストア説明のコピー、軽い言い換えだけの転載、未確認の AI 生成説明
- 実在しない機能、利用実績、レビュー、受賞、数字、更新状況を記載すること
- 広告・提供・アフィリエイト等の関係を隠すこと。該当リンクは関係を明示し、必要に応じて `rel="sponsored"` または `rel="nofollow"` を使う。

## 5. リンク設計

- 各カードの主リンクは、App Store、実際に使える公式サイト、またはプロジェクトの一次情報へ向ける。
- 「詳細はこちら」のような曖昧な文言だけでなく、リンク先が分かる自然で簡潔なアンカーを使う。
- 同じ URL への複数リンクは、画像、名称、CTA など利用者の操作性に明確な役割がある場合だけ許可する。
- 個別ページを作る場合は、トップから到達でき、個別ページから一覧へ戻れる通常の閲覧階層にする。
- 関連プロジェクトへのリンクは、訪問者にとって比較や次の行動に役立つ場合だけ追加する。「SEO のための相互リンク」は禁止する。
- 通常のリンクは `<a href="...">` で実装し、リンク画像には内容を表す `alt` を付ける。
- 自サイトが所有するプロジェクトへの通常の編集リンクに、一律で `nofollow` を付ける必要はない。ただし広告・有償掲載・対価を伴うリンクは適切に区別する。

## 6. コンテンツ品質

- 説明は、用途、対象者、固有の特徴が分かるように書く。宣伝文句だけにしない。
- スクリーンショットは実際の現在の画面を使い、装飾目的でない画像には適切な `alt` を付ける。
- 英語版と日本語版は直訳を強制せず、事実と意図を揃えた自然な文章にする。
- 同種プロジェクトが増えたらカテゴリや絞り込みを改善する。品質を落として件数だけを増やさない。
- 廃止・非公開・大幅変更されたプロジェクトは、リンク切れのまま放置せず、状態表示、更新、または削除を判断する。

## 7. 実装後の検証

- [ ] `git status` と diff を確認し、依頼と無関係な変更を含めていない。
- [ ] 英語版と日本語版の名称、事実、URL、カテゴリ、提供状態が一致している。
- [ ] 追加した全 URL が正常に開き、最終到達先が想定どおりである。
- [ ] リンク文言だけを読んでも、おおよその遷移先が分かる。
- [ ] 画像が表示され、`alt` が正確で、幅・高さ指定等により大きなレイアウト崩れを起こさない。
- [ ] モバイル幅とキーボード操作でカードやリンクを利用できる。
- [ ] HTML の構造、canonical、hreflang、構造化データを壊していない。
- [ ] ページ追加時は、トップから通常リンクで到達でき、重複 canonical や意図しない index 対象を作っていない。
- [ ] 不自然なキーワード反復、隠し要素、過剰リンク、根拠のない主張がない。
- [ ] 新規ページが、リンクを除いても独自の利用者価値を持つ。

検証できなかった項目は、完了報告で明示する。

## 8. 判断に迷った場合

掲載数に固定上限は設けない。件数ではなく、一件ごとの実在性、独自性、利用者価値、保守可能性で判断する。迷う場合は、薄い個別ページを作るより、十分な情報が揃うまで一覧カードに留めるか追加を保留する。
